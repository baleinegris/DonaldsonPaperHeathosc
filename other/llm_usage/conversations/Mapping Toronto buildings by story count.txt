Claude: You just need to change one line — the `cmap` assignment. Here are some good options for storey count data:

**Sequential (best for ordered data like storeys):**
```python
cmap = plt.cm.YlOrRd       # yellow → red (your current)
cmap = plt.cm.viridis       # purple → yellow (perceptually uniform, colorblind-safe)
cmap = plt.cm.plasma        # purple → orange → yellow
cmap = plt.cm.magma         # black → purple → yellow
cmap = plt.cm.Blues         # white → dark blue (clean, print-friendly)
cmap = plt.cm.YlGnBu        # yellow → green → blue
```

**If you want to emphasize extremes:**
```python
cmap = plt.cm.RdYlGn        # red → yellow → green (low → high)
cmap = plt.cm.coolwarm      # blue → red (diverging, good if you have a meaningful midpoint)
```

**Reversing any colormap** (e.g. dark = low instead of high):
```python
cmap = plt.cm.viridis_r     # just append _r to any name
```

For a paper, `viridis` or `Blues` are generally the safest choices — `viridis` is perceptually uniform (equal visual steps per data step), renders well in greyscale, and is colorblind-friendly. `Blues` is very clean and low-distraction on a light basemap.

To apply any of them, just swap that single line in your code:

```python
# Change this line only
cmap = plt.cm.viridis   # or Blues, plasma, magma, etc.
```

Everything else — `vmin`, `vmax`, colorbar, basemap — stays exactly the same.

- **Size legend** uses empty scatters to show what bubble sizes correspond to — pick representative values that make sense for your actual unit range.

Claude: Here's the code for three side-by-side subplots, one per era, using a single consistent colour per panel:

```python
import pandas as pd
import matplotlib.pyplot as plt
import matplotlib.colors as mcolors
import contextily as ctx
import geopandas as gpd

df = pd.read_csv("../data/02-analysis_data/analysis_data.csv")

def to_web_mercator(subset):
    return gpd.GeoDataFrame(
        subset,
        geometry=gpd.points_from_xy(subset.LONGITUDE, subset.LATITUDE),
        crs="EPSG:4326"
    ).to_crs(epsg=3857)

preWarDf  = df[df["YEAR_BUILT"] < 1945]
postWarDf = df[(df["YEAR_BUILT"] >= 1945) & (df["YEAR_BUILT"] < 2000)]
modernDf  = df[df["YEAR_BUILT"] >= 2000]

datasets = [
    (to_web_mercator(preWarDf),  "Pre-War (< 1945)",       "#2166ac"),
    (to_web_mercator(postWarDf), "Post-War (1945–1999)",    "#d6604d"),
    (to_web_mercator(modernDf),  "Modern (2000–)",          "#4dac26"),
]

# Shared extent — compute from full dataset so all panels show the same area
full_gdf = to_web_mercator(df)
xmin, ymin, xmax, ymax = full_gdf.total_bounds
pad_x = (xmax - xmin) * 0.04
pad_y = (ymax - ymin) * 0.04

fig, axes = plt.subplots(1, 3, figsize=(14, 5))

for ax, (gdf, title, colour) in zip(axes, datasets):
    units = gdf["CONFIRMED_UNITS"]
    bubble_size = (units / df["CONFIRMED_UNITS"].max()) * 250  # same scale across panels

    ax.scatter(
        gdf.geometry.x, gdf.geometry.y,
        s=bubble_size,
        c=colour,
        alpha=0.6,
        edgecolors="white",
        linewidths=0.3,
        zorder=3
    )

    ctx.add_basemap(ax, source=ctx.providers.CartoDB.PositronNoLabels, zoom=10)

    # Lock all panels to the same extent
    ax.set_xlim(xmin - pad_x, xmax + pad_x)
    ax.set_ylim(ymin - pad_y, ymax + pad_y)

    ax.set_title(title, fontsize=9, pad=6)
    ax.set_axis_off()

# Shared size legend (units) — attach to rightmost axis
for units_val in [50, 150, 300]:
    axes[-1].scatter([], [], s=(units_val / df["CONFIRMED_UNITS"].max()) * 250,
                     c="grey", alpha=0.6, edgecolors="white", linewidths=0.3,
                     label=f"{units_val} units")
axes[-1].legend(title="Units", title_fontsize=7, fontsize=7,
                loc="lower right", frameon=False)

plt.tight_layout(pad=0.5)
plt.savefig("toronto_eras.pdf", dpi=300, bbox_inches="tight")
plt.show()
```

Key decisions worth noting:

- **Shared extent** — all three panels are locked to the same bounding box (derived from the full dataset), so spatial comparisons between eras are honest. Without this, each panel would auto-zoom to its own points.
- **Shared bubble scale** — `units.max()` is taken from the full `df`, not each subset, so a 300-unit building looks the same size across all three panels.
- **One colour per era** — blue/red/green are distinct enough to tell apart if the figure is ever referenced together, while staying neutral without a gradient.
- **Single legend** — placed on the rightmost panel only to avoid repetition. If your journal requires it on each panel, move the legend block inside the loop.

User: how can I include an access token for basemaps that need it?

Claude: Pass it via the `source` URL directly using an f-string:

```python
TOKEN = "your_token_here"

ctx.add_basemap(
    ax,
    source=f"https://api.mapbox.com/styles/v1/mapbox/light-v11/tiles/{{z}}/{{x}}/{{y}}?access_token={TOKEN}",
    zoom=10
)
```

For providers that have a built-in token field in contextily, you can set it on the provider object instead:

```python
TOKEN = "your_token_here"

# Mapbox
source = ctx.providers.MapBox.Satellite
source["accessToken"] = TOKEN

# Thunderforest (uses apikey instead)
source = ctx.providers.Thunderforest.Atlas
source["apikey"] = TOKEN

ctx.add_basemap(ax, source=source, zoom=10)
```

To find the right key name for any provider, just inspect it:

```python
print(ctx.providers.Thunderforest.Atlas)
# {..., 'apikey': '<insert your api key here>', ...}
```

Whatever placeholder key appears in the output (e.g. `apikey`, `accessToken`, `key`) is what you override.

For cleanliness in a project, store the token in a `.env` file and load it rather than hardcoding it:

```python
import os
from dotenv import load_dotenv

load_dotenv()
TOKEN = os.getenv("MAPBOX_TOKEN")
```

`CartoDB.PositronNoLabels` is one of the few providers that needs no token at all, which is part of why it's a good default for papers.
